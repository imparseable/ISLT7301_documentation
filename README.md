# ISLT7301_documentation
Documentation for the Spring 2026 semester of MU's ISLT 7301 class.

## Who should use this branch for commits
Note that this is a *branch* from the **main** repository. It is meant for contribution by the Spring 2026 section of ISLT7301. It will remain **public** until Monday, March 9th, 2026 to allow folks who didn't submit their usernames to find and request to join.

## What to include in this branch
This branch is for documenting our exploration of issues creating LAMP servers in Spring 2026. Open source code is live and changes, and between the time any issue has arisen and a solution has been found, the variables at play can be quite complex. What we will create is a **snapshot** of our experiences with different **versions** of Ubuntu, Apache2, MySLQ, and PHP as they interact for using WordPress, Omeka, and Koha.

### Where to include it
Please refer to the [ISSUES.md](https://github.com/imparseable/ISLT7301_documentation/blob/Spring2026/ISSUES.md) file to see if your issue is described, or to add an issue.

#### Complete tutorials
Tutorials are detailed guides on how to do something, including checking settings, completing tasks, and solving problems. If you have a tutorial, it goes here. The example tutorial, written by Dr. Ridenour is in the [/tutorials/](https://github.com/imparseable/ISLT7301_documentation/tree/Spring2026/tutorials) directory named [SAMPLE.md](https://github.com/imparseable/ISLT7301_documentation/blob/Spring2026/tutorials/SAMPLE.md).

#### How-to guides
How-to guides are recipes for accomplishing a goal, such as troubleshooting or problem solving. A lot of our book has been how-to guides. In this repository, they are stored in [/how-to_guides/](https://github.com/imparseable/ISLT7301_documentation/tree/Spring2026/how-to_guides), and the corresponding [/how-to_guide/README.md](https://github.com/imparseable/ISLT7301_documentation/blob/Spring2026/how-to_guides/README.md) is about what they look like. Odds are, what you contribute will be a how-to guide.

#### Explanations
Explanations are about the why of the thing. That *aha!* moment when you've made a connection. See [ABOUT_EXPLANATIONS.md](https://github.com/imparseable/ISLT7301_documentation/blob/Spring2026/explanations/ABOUT_EXPLANATIONS.md)

## How to format your work
As with all git pages, the default formatting is Markdown. I recommend the [Markdown Guilde's Markdown Cheat Sheet](https://www.markdownguide.org/cheat-sheet/) as a quick reference.

### When creating a page be sure to include
1. The version of each program you are using (e.g. `php -v`)
2. A description of the problem you are experiencing. If you are using screenshots, please anonymize any personally identifiable information (including your gCloud ID!).
3. The solution, including *in-text links* with descriptive text, (e.g. [the WordPress and PHP compatibility page recommends PHP version 8.3 minimum for WordPress versions 6.8 and 6.9](https://make.wordpress.org/core/handbook/references/php-compatibility-and-wordpress-versions/)).
4. Any possible other solutions based on what you learned from fixing a problem.
5. Any other information that is relevant, or required to fix the problem.

Including `code` is helpful, and full paths *and* abbreviated paths may be used to clarify where in the directory structure and which files or settings were changed.
